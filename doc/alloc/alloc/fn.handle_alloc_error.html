pub fn handle_alloc_error(layout: Layout) -> !
Expand description

由于内存分配错误或失败而终止。

鼓励希望响应分配错误而终止计算的内存分配 API 调用程序调用此函数，而不是直接调用 panic! 或类似方法。

该函数的默认行为是将一条消息打印到标准错误并终止该进程。 可以将其替换为 set_alloc_error_hook 和 take_alloc_error_hook。
